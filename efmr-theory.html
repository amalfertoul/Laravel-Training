Exercice 2: (6 pts)

Q1) Quelle est la différence entre les migrations et les seeders dans Laravel? (2pts)

Réponse: Les migrations sont utilisées pour gérer le versionnement de la structure de la base de données (créer, modifier, supprimer des tables et des colonnes). Les seeders sont utilisés pour insérer des données initiales ou de test dans la base de données.

Q2) Comment définir une relation "One-to-Many" entre deux modèles dans Laravel? (2pts)

Réponse: Dans le modèle parent, utilisez la méthode `hasMany()`. Dans le modèle enfant, utilisez la méthode `belongsTo()`.

Q3) Qu'est-ce que les middlewares dans Laravel et à quoi servent-ils ? (2pts)

Réponse: Les middlewares sont des filtres qui inspectent et/ou filtrent les requêtes HTTP entrant dans votre application. Ils servent à exécuter des tâches avant ou après le traitement d'une requête, comme l'authentification, la journalisation, la gestion des sessions, etc.
